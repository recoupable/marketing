Some interesting ideas from the plugin:


There is a /testimonial skill that pulls info about the customer from their agents memory, then writes a testimonal from them and appends it to nurijanian's google sheet.
There is a context gate, that blocks workflows from running if the user has not provided enough context.
There is a tidy skill that sets up a scheduled task to clean the workspace daily from stale files and data. 
It assumes the user will be running everything out of one specific local directory (and scaffolds the workspace accordingly on first run).
