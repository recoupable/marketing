# Vibe Check: Cursor 3.0 Bets Big on Agent Orchestration 

*The AI-native IDE is now becoming an agent-orchestration tool. Will it work?*

There&rsquo;s a particular awkwardness when a well-loved product stops being what it was, but hasn&rsquo;t yet become what it wants to be. Cursor 3.0&mdash;a totally rebuilt version of Cursor centered around agent-orchestration&mdash;out today, is in that weird in-between stage: There are many promising early signs, but it&rsquo;s not yet mature enough to merit a switch.

Cursor helped create the [AI-native integrated development environment (IDE)](https://every.to/vibe-check/vibe-check-cursor-2-0-and-composer-1-alpha). It was the product that made &ldquo;coding with AI in your editor&rdquo; feel like a genuine category. But the thing that made them a category leader has turned out to be constraining.

The rise of [Claude Code](https://every.to/podcast/how-to-use-claude-code-like-the-people-who-built-it) shifted the dominant coding paradigm away from pair programming&mdash;which requires a code editor&mdash;toward agent delegation and orchestration&mdash;which doesn&rsquo;t. That left Cursor in a tough spot.

Do they satisfy their core IDE audience and continue building on top of an editor? Or do they start from scratch and build an agent-orchestration tool like Claude Code, Codex, and Conductor?

With Cursor 3.0, they&rsquo;ve decided to take the plunge toward the latter: This release is a rebuilt product made for dispatching, monitoring, and managing AI agents rather than manually writing code. We&rsquo;ve been testing it internally for about a week, and there are a lot of promising signs. It&rsquo;s extremely fast and resource-light, it has an impressive local-to-cloud programming system, and its [built-in model](https://every.to/vibe-check/vibe-check-cursor-2-0-and-composer-1-alpha), Composer 2, is quick, smart, and concise.

But as a product, it&rsquo;s still too early in its lifecycle to be a serious contender against the more mature agent orchestration offerings from Anthropic and OpenAI.

The question that kept coming up during our testing was, &ldquo;Who is this for?&rdquo; Power users who already work in Claude Code or Codex won&rsquo;t use a new agent orchestration layer unless it&rsquo;s 10 times better than what exists, because they already have one that works well. And existing Cursor users who love the IDE are likely to be upset that it has been deprioritized.

That&rsquo;s why we believe Cursor is in an awkward in-between stage. They&rsquo;ve made the right strategic move to prioritize agent orchestration. But it will take some time before what they have is truly competitive. Their team seems to be iterating incredibly fast, so we&rsquo;ll be paying attention over the coming weeks and months as it improves.

## What&rsquo;s new

Cursor 3.0 features a new default interface built from scratch&mdash;a departure from the VS Code fork that the product was originally built on. The company says it keeps the most useful pieces from earlier versions (the browser, diffs, file, and plan viewing) while adding support for patterns that reflect how coding with AI has changed. Highlights according to the Cursor team:

Before

Editor with AI

Now

AI with an editor

- Cloud and local agents that can hand tasks back and forth

- Cloud agents that run on their own machines

- Working across multiple workspaces

- Agent-first git management, or keeping track of changes to a project&rsquo;s files

- A new model picker that lets users set defaults based on their preferences, with access to any frontier model, including Cursor&rsquo;s own Composer 2

The Cursor team notes that the editor is still present, just no longer center stage. In practice, the shift is bigger than that framing suggests. The default view is an orchestration panel&mdash;repos on the left, a chat-driven workflow in the center, browser and markdown editors docked alongside. The file explorer is gone from the default layout. Where Cursor used to be an editor with AI built in, it&rsquo;s now an AI dashboard with an editor tucked behind it.

Cursor charges a base price per seat. But on top of that, some AI usage is metered&mdash;the more you use it, and the more expensive the model you pick, the more you may end up paying. [Kieran Klaassen](https://every.to/@kieran_1355) said he spent roughly $2,000 in the course of testing (comped by the Cursor team). For heavy users, the cost may end up looking very different from the sticker price.

## The headline findings

Finding 01

### The identity gap shows up in missing basics

The identity gap&mdash;it can&rsquo;t decide if it&rsquo;s an IDE or an agent orchestration tool&mdash;shows up in small, concrete ways. The filesystem sidebar has been demoted to another tab. You can&rsquo;t be an IDE without file navigation more easily available&mdash;developers need to see and browse their project structure to orient themselves in a codebase.

And you can&rsquo;t be a serious agent orchestrator if developers still need GitHub Desktop to create a branch, because branching is how you give an agent a safe sandbox to work in without risking your main code.

Missing as an IDE

No filesystem sidebar, no file explorer, no branch selector or creation.

Missing as an orchestrator

Sessions don&rsquo;t persist, can&rsquo;t open agents side by side, no branch sandboxing.

&ldquo;I can&rsquo;t get over not having the filesystem on the left. And relegating it to another tab breaks up my workflow.&rdquo;

&mdash;Mike Taylor

Finding 02

### The harness fights the model

The harness&mdash;the layer of software that connects the AI model to Cursor&rsquo;s interface and controls how it behaves&mdash;is too eager to jump into writing code. Kieran asked Opus 4.6 to investigate a bug, and it immediately started rewriting code instead of looking into the problem first.

In one session, it deleted his entire Codex skill folder. He doesn&rsquo;t see this behavior in Claude Code with the same model, which means that the issue is Cursor&rsquo;s harness, not the AI itself. Skills load separately from Claude, Codex, and Cursor with no deduplication.

&ldquo;It should not disappear like it never existed.&rdquo;

&mdash;Kieran Klaassen, on session persistence

What works

### Desktop app performance

The first thing we noticed is that Cursor really knows how to build a desktop application. **Andrey Galko**&rsquo;s first reaction was &ldquo;faster, better app,&rdquo; and that baseline performance makes a difference when you&rsquo;re spending all day in a tool.

### Composer 2 earns its spot

Composer 2&mdash;[Cursor&rsquo;s own LLM](https://every.to/vibe-check/vibe-check-cursor-2-0-and-composer-1-alpha)&mdash;earns its spot as the default model. Multiple testers noted it&rsquo;s concise, fast, and good at telling you the most salient information succinctly&mdash;an improvement over models that bury the answer in three paragraphs of context. The app itself is also faster.

### Local-to-cloud orchestration

Local-to-cloud orchestration is the standout feature of this release. The workflow goes like this: You start a task on your machine, decide it&rsquo;s going to take a while, and push it to a cloud virtual machine with one click. The agent keeps working on its own&mdash;writing code, running tests, building the feature&mdash;while you close your laptop or move on to something else. When it&rsquo;s done, it sends you a screencast: a video of itself demoing the feature it just built, navigating the user interface, and clicking through flows.

### Agentic orchestration as default

The bigger structural bet&mdash;making agentic orchestration the default mode&mdash;is one to keep an eye on. If you&rsquo;ve never used Claude Code from a terminal, set up worktrees manually, or built your own prompt workflows, Cursor 3.0 guides you with its visual interface for all of that. Whether that grammar is clear enough for existing Cursor lovers to adopt this new way of working is another question&mdash;one we&rsquo;ll come back to.

What needs work

### The identity gap shows up in missing basics

The identity gap&mdash;it can&rsquo;t decide if it&rsquo;s an IDE or an agent orchestration tool&mdash;shows up in small, concrete ways. The filesystem sidebar has been demoted to another tab. You can&rsquo;t be an IDE without file navigation more easily available&mdash;developers need to see and browse their project structure to orient themselves in a codebase. And you can&rsquo;t be a serious agent orchestrator if developers still need GitHub Desktop to create a branch, because branching is how you give an agent a safe sandbox to work in without risking your main code.

Mike&rsquo;s reaction was visceral: &ldquo;I can&rsquo;t get over not having the filesystem on the left. And relegating it to another tab breaks up my workflow. I have spent over eight hours a day for years looking to the left to navigate, so this is not a small change for me.&rdquo; Kieran couldn&rsquo;t tell what branch he was on without asking the model, and the UI showed commit buttons on sessions with nothing to commit.

Cursor uses its own .cursor folder rather than interoperating with Claude Code&rsquo;s skills format, so anyone using both tools is forced to maintain two separate systems. The Claude .skill files don&rsquo;t render as markdown, which makes browsing and editing them painful.

![](/assets/cursor-vibecheck/skills-triplication-c9261db1ee7f43fa9a1a90836d2d314599a6e993764650d84b2cbb44961efb1a.jpg)

Between Claude Code, Codex, and Cursor, Kieran has to maintain three separate sets of skills. (Image courtesy of Kieran Klaassen.)

### The harness fights the model

The harness&mdash;the layer of software that connects the AI model to Cursor&rsquo;s interface and controls how it behaves&mdash;is too eager to jump into writing code. Kieran asked [Opus 4.6](https://every.to/vibe-check/opus-4-6) to investigate a bug, and it immediately started rewriting code instead of looking into the problem first. In one session, it deleted his entire [Codex](https://every.to/podcast/how-openai-s-codex-team-uses-their-coding-agent) skill folder. He doesn&rsquo;t see this behavior in Claude Code with the same model, which means that the issue is Cursor&rsquo;s harness, not the AI itself. Skills load separately from Claude, Codex, and Cursor with no deduplication&mdash;and in Kieran&rsquo;s case, a duplicate marketplace install made it worse.

### Session persistence is fragile

Session persistence&mdash;the app&rsquo;s ability to remember what you were doing when you switch between tasks&mdash;is fragile. Kieran connected to a remote server, selected a folder, and started working. When he switched to another agent chat inside Cursor to check his daily plan and came back, the connection was gone&mdash;as if he&rsquo;d never set it up. &ldquo;It should not disappear like it never existed,&rdquo; he said. If the whole point of the product is coordinating multiple agents at once, you need to be able to move among them without losing your place.

![](/assets/cursor-vibecheck/ssh-navigation-10f58fae39b30ab6b0c23b087b71a6344635cbe400174721be292ec2dc588d3f.jpg)

Basic remote navigation feels underexplained: From an active Secure Shell (SSH) session, where you are remotely controlling another computer, it&rsquo;s not obvious how to switch to another folder without needing to reconnect from scratch. (Image courtesy of Kieran Klaassen.)

### Inconsistent interaction model

The interaction model&mdash;how you tell the agent what to do&mdash;is inconsistent. Sometimes you click buttons, at others you type in the chat window, and the two don&rsquo;t feel unified. Kieran tried to open multiple agent tabs side by side, something he does routinely with terminal windows, and couldn&rsquo;t figure out how after five minutes. He couldn&rsquo;t see what branch of the code he was working on without asking the agent.

### Composer 2&rsquo;s speed cuts both ways

Composer 2&rsquo;s speed cuts both ways. Kieran agreed it&rsquo;s snappy but added that it&rsquo;s &ldquo;not as deep&rdquo;&mdash;fine for surface-level tasks, but lacking the depth of [Opus 4.6](https://every.to/vibe-check/opus-4-6) for complex reasoning. And smaller user experience issues add friction&mdash;a notification sound on every message, all repos visible in the sidebar even after selecting a project (Mike: &ldquo;one repo = one context window&rdquo;&mdash;he wants to be able to focus on one project at a time), no GitHub issues tab alongside pull requests.

Quick verdict

### For IDE users

Missing file navigation and branch management make it hard to use as a primary editor.

### For agentic engineers

Underpowered compared to Claude Code and Codex. Session persistence and the harness layer need work.

### For newcomers to AI coding

The visual interface is promising, but the interaction model confused even power users.

## The bottom line

Cursor 3.0 is a bet that the future of coding is managing agents. The layout, the speed improvements, and Composer 2&rsquo;s conciseness all point toward a product that&rsquo;s thinking seriously about what comes after the IDE.

But the growing pains inherent in this transition are showing. Basic features are missing, the skills ecosystem is fragmented, and power users are mostly shrugging. Mike framed it as a question of management style: Are you the person who wants to dig into every line, or the one who sets a vision and lets the system execute? Cursor is picking the delegation side&mdash;and that may well be the right call. The problem is that Claude Code and Codex have already established themselves in the delegation market. Building the best IDE with AI built in was a defensible position&mdash;Cursor had a genuine edge there.

Kieran initially hoped Cursor 3.0 would open agentic engineering to a wider audience. After a morning of testing, he reversed course:

&ldquo;Even being an agentic engineer using Cursor feels confusing. I don&rsquo;t think this will inspire millions of people to start coding like this.&rdquo;

![](/assets/team/kieran-bb3200155bbd5b326676eb4cc57fd55d69172efcca3d92ee214796ef7be77a6e.png)

Kieran Klaassen

If a power user who lives in Claude Code can&rsquo;t find his footing, the newer audience Cursor is betting on faces an even steeper climb.

Cursor is no longer competing to be the best IDE. It&rsquo;s competing to be the best way to talk to an AI that writes code for you&mdash;and that&rsquo;s a much more crowded race.
