# ARTICLES.md — Content Concept Backlog

Running list of article/content concepts, each upgraded to its best shippable version. Format:

```
**Hook:**          ← the working title / headline (stands alone to a cold reader; outcome-led where possible)
**Description:**   ← what the article covers, including the takeaway artifact it ships
**Value:**         ← one sentence: WHO it's for + HOW it helps them
**Deliverable:**   ← the concrete free artifact to ship + the funnel CTA (this is the lever made real)
**Value Score:**   ← 1–5 (see scale); set to the realistic ceiling for this concept
**From:**          ← internal sourcing only (never surfaced in the published piece)
```

**Value Score scale:** 5 = flagship; broad reach, strong proof, dead-on strategy. 4 = strong, clear single audience, real receipts. 3 = solid but niche (its honest ceiling). 2 = needs fact-clearing first. 1 = weak/risky.

**The funnel the deliverables run (from the `referances/` teardowns).** Every reference company (Every, pm-world, Tenex) runs the same move: *teach something genuinely useful, and make the teaching double as proof, product, and positioning.* The content never opens with "buy our thing" — it opens with a lesson, and the lesson sets up the product or the consulting conversation. The levers each `Deliverable:` draws on:
- **L1 — Ship a takeaway artifact** (template / checklist / prompt-pack / calculator). A read becomes a saved tool → SEO, shares, lead magnet. *(Every's guides, Tenex's playbooks, pm-world's free prompts.)*
- **L2 — Embed the product as the payoff** ("…or the skill does this in one command"). *(pm-world: every post is the product changelog.)*
- **L3 — Lead with the outcome + real receipts.** *(pm-world's outcome headlines; Tenex's specific-mechanic testimonials.)*
- **L4 — Use our own label as proof** (real Gatsby Grace numbers — unfakeable). *(Every's "how we run a company on agents.")*
- **L5 — Coin the vocabulary** so readers adopt our word for it. *(Tenex's "AI-absent → AI-native.")*
- **L6 — End on the right funnel rung:** free `npx skills add recoupable/skills` → newsletter/membership → qualified "run this for your roster" label call.
- **L7 — Earn the expert conversation:** frame so a label finishes reading thinking "I want them to run this for us."

**Editorial rules:**
- **One reader per article** (sections are grouped by audience).
- **No naked jargon in a hook;** expand acronyms once; cut tool names from titles unless the audience is engineers.
- **No internal references in reader-facing text** (PR/issue IDs, file paths, codenames stay in `From:`).
- **⚠ = a pre-publish gate:** state the timeframe/subject and confirm it's cleared to share before shipping.
- **Name competitors only on purpose:** Every / pm-world / Tenex are our reference teardowns — name + link them only when the article is explicitly about them.

**Glossary (defined once):**
- **Gatsby Grace** — the artist we run in-house as our live testbed; most "build-in-public" pieces are her campaigns.
- **"Dead" / retired song** — a song with low engagement we've stopped actively promoting (vs. a proven winner we lean into).
- **Skill** — an installable AI-agent capability pack; users add ours with `npx skills add recoupable/skills`.
- **Scroll-stopper (video hook)** vs. **the musical hook (best 15s of a song)** — two different "hooks"; never use the bare word for both.
- **The reference trio** — Every, pm-world, Tenex: companies whose funnels we tore down as models for selling AI expertise.
